# 上线后检查清单

这份清单适合在站点第一次上线，或者每次改完重要配置后快速回归检查。

## 1. 页面是否能正常打开

至少检查下面这些地址：

- `/`
- `/posts/`
- `/tags/`
- `/search/`
- `/about/`
- `/index.xml`
- 一篇文章详情页
- 一个标签详情页
- 一个不存在的路径，确认 404 页面正常

## 2. 内容和交互是否正常

- 中文内容没有乱码
- 首页和列表页卡片排版正常
- 搜索能返回结果，也能正确显示无结果状态
- 分页链接、标签链接、导航链接都可点击
- 文章页目录、代码块复制按钮、上一篇/下一篇都正常

## 3. SEO 与分享信息是否正常

在浏览器开发者工具里，或者直接查看页面源码，确认：

- `canonical` 指向正确域名
- `og:url`、`og:image`、`twitter:image` 不是占位值
- `title` 和 `description` 与页面内容匹配
- RSS 地址是可访问的

## 4. 域名与缓存是否正常

如果你已经绑定自定义域名：

- 自定义域名可以访问
- `*.pages.dev` 是否需要保留访问，还是要重定向到正式域名
- 如果发现新内容不刷新，先排查额外的缓存规则或 Page Rules

## 5. 评论与讨论是否正常

如果你准备开启 giscus：

- GitHub 仓库已经启用 Discussions
- giscus app 已安装到仓库
- 文章页能正常显示评论框
- 首次评论后 GitHub Discussions 中会自动创建讨论串

## 6. 发布后建议保留一次记录

建议至少留一条简单记录，方便以后排查：

- 上线日期
- 当前正式域名
- 当前 Cloudflare Pages 项目名
- 是否已开启 giscus
- 是否已做 pages.dev 到正式域名的重定向
